## 1. ВИЗУАЛЬНАЯ ИЕРАРХИЯ

### Сильные стороны:
- Чёткий Hero-блок с главным заголовком
- Хорошее использование цвета для акцента (синий #0071E3)
- Видимые CTA-кнопки

### Проблемы:
1. **Hero-блок перегружен** — badge, заголовок, подзаголовок, CTA, таймер, 3 иконки — слишком много элементов на первом экране
2. **Таймер отвлекает** — красный цвет таймера конкурирует с CTA-кнопкой
3. **Социальное доказательство после Hero** — цифры 3500+ не так впечатляют, как реальные отзывы

### Улучшения:
```
ДО: Badge → H1 → Subtitle → CTA → Таймер → 3 иконки
ПОСЛЕ: Badge → H1 → Subtitle → CTA → 2 иконки (убрать таймер из Hero)
```

---

## 2. КОМПОЗИЦИЯ

### Сильные стороны:
- Чистый минималистичный дизайн
- Единая цветовая палитра
- Хорошие отступы между секциями

### Проблемы:
1. **Нет визуального якоря** — Hero-блок не привлекает внимание к главному
2. **Карточки проблем и преимуществ выглядят одинаково** — нет визуального различия
3. **Нет фотографий** — сайт выглядит «дешёво» без реальных изображений

### Улучшения:
- Добавить.hero-фото (человек, который говорит)
- Сделать карточки проблем более «болезненными» (красные акценты)
- Добавить фотографии преподавателей и учеников

---

## 3. ЧИТАЕМОСТЬ

### Сильные стороны:
- Хороший контраст текста (серый на белом)
- Размер шрифтов соответствует иерархии
- Межстрочный интервал 1.6 — удобно читать

### Проблемы:
1. **Подзаголовки слишком светлые** — #86868B на белом фоне может быть сложно для людей со слабым зрением
2. **Длинные абзацы в кейсах** — сложно воспринимать большие блоки текста
3. **Нет выделения ключевых слов** — глаз не цепляется за главное

### Улучшения:
- Увеличить контраст подзаголовков до #6E6E73
- Разбить длинные абзацы на bullet points
- **Жирным** выделить ключевые цифры и результаты

---

## 4. МОБИЛЬНАЯ ВЕРСИЯ

### Сильные стороны:
- Адаптивная сетка
- Sticky CTA внизу экрана
- Гамбургер-меню

### Проблемы:
1. **Hero-блок слишком высокий** на мобильном — занимает весь экран, CTA не виден без скроллинга
2. **Карточки тарифов** — выделенный тариф не выглядит привлекательнее на мобильном
3. **FAQ аккордеон** — иконка «вниз» не интуитивна (ожидается «вправо»)
4. **Форма в финальном CTA** — поля слишком большие, кнопка «тонет»

### Улучшения:
```css
/* Hero на мобильном */
.hero {
    min-height: auto;
    padding: 100px 0 40px; /* Уменьшить отступы */
}

/* Кнопка CTA на мобильном */
.sticky-cta .btn {
    padding: 14px 24px;
    font-size: 14px;
}
```

## 5. ЛОГИКА БЛОКОВ

### Сильные стороны:
- Последовательность: Проблемы → Решение → Кейсы → Отзывы → Тарифы → FAQ → CTA
- Каждый блок отвечает на вопрос клиента

### Проблемы:
1. **Нет блока «Как это работает»** перед тарифами — клиент не понимает процесс
2. **Преподаватели после отзывов** — логичнее показать их раньше (до кейсов)
3. **FAQ перед финальным CTA** — клиент может «застрять» в вопросах и не дойти до формы

### Улучшения:
```
Текущий порядок:
Hero → Соцдоказательство → Проблемы → Преимущества → Как это работает → Кейсы → Отзывы → Преподаватели → Тарифы → FAQ → CTA → Контакты

Оптимальный порядок:
Hero → Соцдоказательство → Проблемы → Преимущества → Как это работает → Преподаватели → Кейсы → Отзывы → Тарифы → FAQ → CTA → Контакты
```

## 6. КОНВЕРСИОННОСТЬ

### Сильные стороны:
- Чёткий CTA «Записаться на бесплатный урок»
- Гарантия возврата денег
- Таймер обратного отсчёта

### Проблемы:
1. **Таймер не создаёт реальной срочности** — он сбрасывается каждые 24 часа
2. **Нет ограничения мест** — «Осталось 5 мест» не подтверждается
3. **Форма слишком простая** — нет полей для целей обучения
4. **Нет live-чата** — клиент не может задать вопрос мгновенно
5. **Нет повторных CTA** — кнопка «Записаться» только в Hero и финале

### Улучшения:
- Добавить CTA после каждого блока («Узнать больше», «Посмотреть тарифы»)
- Добавить live-чат или WhatsApp-кнопку
- Сделать таймер реальным (привязать к акции)

---

## 7. ДОВЕРИЕ

### Сильные стороны:
- Отзывы с именами и возрастом
- Гарантия возврата денег
- Кейсы с конкретными результатами

### Проблемы:
1. **Нет реальных фотографий** — аватары с буквами выглядят фейково
2. **Нет ссылок на VK/Яндекс** — невозможно проверить отзывы
3. **Нет лицензии** — для образовательных услуг это важно
4. **Нет видео** — видео-отзывы вызывают больше доверия
5. **Нет логотипов компаний** — нет социального доказательства

### Улучшения:
- Добавить реальные фото (или хотя бы стоковые)
- Ссылки на VK/Яндекс для проверки отзывов
- Добавить номер лицензии
- Добавить 1-2 видео-отзыва

---

## 8. УДОБСТВО ЗАЯВКИ

### Сильные стороны:
- Простая форма (3 поля)
- Чёткая кнопка
- Гарантия под формой

### Проблемы:
1. **Нет валидации телефона** — можно ввести любой текст
2. **Нет маски телефона** — неудобно вводить номер
3. **Нет обработки ошибок** — если сервер недоступен, пользователь не узнает об этом
4. **Нет progress indicator** — непонятно, что происходит после отправки

### Улучшения:
```javascript
// Маска телефона
input[type="tel"] {
    font-family: monospace;
}

// Валидация
phone.addEventListener('input', (e) => {
    const value = e.target.value.replace(/\D/g, '');
    e.target.value = value.replace(/(\d{3})(\d{3})(\d{2})(\d{2})/, '+7 ($1) $2-$3-$4');
});
```

## ЗАКЛЮЧЕНИЕ

Сайт имеет хорошую основу и правильную логику. Основные проблемы связаны с:
1. Отсутствием реальных фотографий
2. Недостаточным доверием (нет лицензии, видео, ссылок)
3. Мелкими UX-проблемами на мобильном

После внесения улучшений сайт может претендовать на оценку 8.5-9/10 по шкале Awwwards.
